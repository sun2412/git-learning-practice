五、Git 学习心得
 
通过本次实践，我掌握了 Git 的基础使用流程，对版本控制有了清晰的理解：
 
1. Git 的核心价值：它就像一个「代码时光机」，每一次  commit  都会生成一个代码快照，不仅能记录修改历史，还能随时回退到任意版本，有效避免了代码丢失和版本混乱的问题。

2. 配置的重要性： user.name  和  user.email  是 Git 识别提交者身份的关键，必须正确配置，否则无法正常提交和推送代码。

3. 规范提交的意义：清晰的  commit message （如  feat / docs / fix  开头）能让后续查看历史时快速了解每次修改的内容，养成规范提交的习惯对团队协作非常重要。

4. 问题解决能力提升：在解决  git config  和  git push  报错的过程中，我学会了通过查看官方文档和报错信息定位问题，提升了独立解决技术问题的能力。
